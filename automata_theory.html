Foundation

What is Automata Theory?

Automata theory studies abstract machines (automata) and the computational problems they can solve. It forms the mathematical backbone of compilers, programming languages, AI, cryptography, and hardware design — asking: "What can a machine compute, and what are its limits?"

THE COMPUTATIONAL POWER LADDER — FROM WEAKEST TO STRONGEST

Automaton (pl. Automata)

An abstract self-operating machine defined by: a set of states, an input alphabet, a transition function, a start state, and accept states. It reads input symbols and changes state according to rules.

Formal Language

A set of strings over an alphabet Σ. E.g., the language of all binary strings with even number of 1s. Automata recognize (accept/reject) languages. Different machines recognize different classes of languages.

L = { w ∈ Σ* | w satisfies property P }

Why it Matters

Compilers use finite automata for lexical analysis, context-free grammars for parsing. Turing machines define the limits of what computers can do. Automata underpin regex engines, XML validators, network protocols, and more.

Level 1 Automata

Deterministic & Nondeterministic Finite Automata

A Finite Automaton has a fixed, finite set of states, reads input one symbol at a time, and either accepts or rejects the string. A DFA has exactly one transition per symbol per state. An NFA may have zero, one, or many.

DFA — Formal Definition

A 5-tuple M = (Q, Σ, δ, q₀, F)

Q — finite set of states
Σ — input alphabet
δ: Q×Σ→Q — transition function
q₀ ∈ Q — start state
F ⊆ Q — set of accept states

Exactly 1 transition per (state, symbol)

NFA vs DFA

An NFA may have 0, 1, or many transitions for a symbol — or even ε (epsilon) transitions that consume no input. It accepts if any possible computation path reaches an accept state. Surprisingly, NFA and DFA recognize the exact same class of languages!

NFA ≡ DFA (same power, different size)

Subset Construction

Every NFA can be converted to an equivalent DFA via the subset construction algorithm. Each DFA state represents a set of NFA states. A DFA may need up to 2ⁿ states to simulate an NFA with n states.

Acceptance

A DFA accepts a string w if, starting from q₀ and following δ for each symbol of w, the machine ends in a state q ∈ F. The language of M is L(M) = all strings accepted by M.

Epsilon (ε) Transitions

An ε-NFA can change state without reading any input symbol. This makes constructing automata from regex much easier. The ε-closure of a state is all states reachable via ε alone. ε-NFAs are also equivalent in power to DFAs.

Pumping Lemma

A tool to prove a language is NOT regular. If L is regular, any long enough string can be "pumped" (middle part repeated any number of times) and still be in L. Example: {aⁿbⁿ} fails the pumping lemma, so it is not regular.

Regular Expressions

Regular Expressions & Their Connection to Finite Automata

Regular expressions are a compact algebraic notation for describing regular languages — exactly the languages recognized by finite automata. Every regex can be converted to an NFA (Thompson's construction), and vice versa.

REGULAR EXPRESSION OPERATORS & NFA FRAGMENTS

Regex Operators

Three primitive operations suffice to build any regex:

· Concatenation — RS (r then s)
| Alternation — R|S (r or s)
* Kleene star — R* (0 or more r)

All other operators (?, +, {n,m}, character classes) are syntactic sugar for these three.

Kleene's Theorem

A language is regular if and only if it can be described by a regular expression. This means:
DFA ⟺ NFA ⟺ ε-NFA ⟺ Regex

All four formalisms are equivalent in power.

Common Regex Examples

a*b* — zero or more a's then b's
(a|b)*abb — any string ending in "abb"
[0-9]+ — one or more digits
(a|ε)b — "b" or "ab"
Σ* — all strings over alphabet

Thompson's Construction

Converts a regex to an ε-NFA in O(n) states. Build NFAs for base cases (single symbols), then combine using the three operators by wiring NFA fragments with ε-transitions as shown in the diagrams.

Real-World Usage

Regex engines power: grep, sed, Python re module, JavaScript RegExp, lexical analysers (flex/lex), search engines, input validation. Understanding NFAs helps you reason about catastrophic backtracking in regex engines that use backtracking simulation.

Level 2 Automata

Pushdown Automata (PDA)

A PDA is a finite automaton augmented with an infinite stack. The stack lets it remember unlimited information, enabling it to recognize context-free languages like { aⁿbⁿ | n≥0 } — impossible for a DFA.

PDA ARCHITECTURE — CONTROL UNIT + STACK
PDA EXAMPLE — RECOGNISING aⁿbⁿ (e.g. "aabb")
CONTEXT-FREE GRAMMAR — EQUIVALENT TO PDA

Why a Stack?

A DFA has no memory beyond its state. A stack gives LIFO memory — perfect for counting, matching brackets, and parsing recursive structures. The stack can hold unlimited symbols, but only the top is accessible at any time.

PDA Formal Definition

A 7-tuple M = (Q, Σ, Γ, δ, q₀, Z₀, F)

Γ — stack alphabet
Z₀ — initial stack symbol
δ: Q×(Σ∪{ε})×Γ → P(Q×Γ*)

A PDA is inherently nondeterministic; deterministic PDAs (DPDA) are strictly less powerful.

CYK Algorithm

Cocke-Younger-Kasami: parses a string using a CFG in O(n³) time. First convert grammar to Chomsky Normal Form (CNF): all rules are A→BC or A→a. Then fill a triangular table bottom-up.

Ambiguity

A CFG is ambiguous if some string has two or more distinct parse trees. Example: E→E+E|E*E|n — the string "1+2*3" has two parse trees. Compilers need unambiguous grammars or precedence rules.

Level 4 Automata

The Turing Machine

A Turing Machine (TM) is the most powerful computational model. It has a two-way infinite tape, a read/write head, and a finite control. A TM can simulate any algorithm — and defines the limits of what is computable at all.

DECIDABILITY — WHAT TURING MACHINES CANNOT DO

TM Formal Definition

A 7-tuple M = (Q, Σ, Γ, δ, q₀, q_acc, q_rej)

Γ — tape alphabet (Σ ∪ {blank})
δ: Q×Γ → Q×Γ×{L,R}

Unlike PDA: head can move LEFT, tape is writable, no separate stack — the tape IS the memory.

The Halting Problem

Does a given TM M halt on input w? Turing proved in 1936 this is undecidable — no TM can solve it for all inputs. Proof by contradiction: assume H solves halting, construct D(M) = H(M,M) with inverted answer — paradox! This is the most fundamental limit of computation.

Church-Turing Thesis

Any effectively computable function can be computed by a Turing machine. This is not a theorem — it's a thesis about the nature of computation. It implies that Python, C, LISP, lambda calculus, cellular automata — are all equally powerful (Turing-complete).

Variants — Same Power

All of these are equivalent to a standard TM:
• Multi-tape TM
• Nondeterministic TM
• TM with 2D infinite tape
• RAM machine / modern CPU
• Lambda calculus
• Conway's Game of Life(!)

Complexity Classes

TMs also model computational difficulty:
P — decidable in polynomial time
NP — verifiable in poly time
NP-complete — hardest in NP
PSPACE, EXP — harder still
P = NP? — the greatest open problem in CS.

The Chomsky Hierarchy

Noam Chomsky classified all formal grammars into four nested levels (Types 0–3), each strictly more expressive than the previous. The hierarchy elegantly unifies grammar theory, automaton theory, and computational complexity.
